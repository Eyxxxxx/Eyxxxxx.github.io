python-安装mxnet库

发表于 2021-03-23 更新于 2021-03-25 分类于 村口修电脑 阅读次数：

环境

win10 64位
Python 3.8.2 (tags/v3.8.2:7b3ab59, Feb 25 2020, 23:03:10) [MSC v.1916 64 bit (AMD64)] on win32
pip 20.2.4 (这个东西的版本在这里应该不重要, 有就行)

注: mxnet有很多装法, 包括基于GPU的、基于CPU和GPU的等, 我应该是配了一个基于CPU的最简单的环境.

问题描述

使用pip install mxnet无法成功安装mxnet模块

重要链接(备份)

  1. PyPI · The Python Package Index
  2. Python Extension Packages for Windows - Christoph Gohlke
  3. windows python库wheel文件下载_聿默的博客-CSDN博客_python wheel下载

太长不看版-结论

  • 使用pip show numpy, 若已有numpy则使用pip uninstall numpy将其卸载
  • 打开上述重要链接中第二个链接. ctrl+f搜索MXNet, 选择合适版本下载. 再ctrl+f 搜索numpy选择1.19.5的合适版本下载.(我没试最新的1.20版)
  • 在下载了上述两个文件的目录下, 使用pip install 下载的文件名.whl进行安装. 例如: pip install numpy-1.19.5+mkl-cp38-cp38-win_amd64.whl
    • 先安装 numpy
    • 再安装 MXNet

详述原地绕两圈后解决问题的过程

注: 本人目前为止没有正经学过任何python知识. 只是奉行拿来主义, 所以底下的思路若显的兜圈子请轻嘲.

  • 使用pip install mxnet出现报错, 根据大红字上面的话初步判定是numpy安装失败导致mxnet安装失败. 于是试图先单独安装numpy.
图一
  • 一开始搜索关键词Building wheel for numpy (setup.py) ... error 没得到有用信息.
  • 改搜numpy 的 wheel文件发现下述第一个参考链接给出的第一个网址PyPI · The Python Package Index, 在其中搜索numpy, 选择下载文件时发现有cp37和cp38, 查阅发现是指python3.7和python3.8. 在cmd运行python -V 发现我的python是3.8.

    参考链接

    1. windows python库wheel文件下载_聿默的博客.
    2. Python下numpy不成功的解决办法（wheel方法安装，试用其他包）_cvMat的博客-CSDN博客
图二
  • 按照第二个参考链接的教程pip install numpy-1.20.1-cp38-cp38-win_amd64.whl, numpy安装成功了, 但是mxnet仍旧安装失败. 眼拙的我终于看到了绿框中的限制条件 (实际上图一中我画的第一个框里就有写=^=.)
图三
  • 然后开始找旧版本的numpy, 惊喜的发现版本小于17.0.0的最高版本是16.6.0, 而该版本没有支持到python 3.8. 不信你看 -> numpy · PyPI
  • 又查了一下numpy的版本问题找到numpy 版本改换_shuijinghua的博客-CSDN博客_numpy版本, 并按教程使用pip install -U numpy==1.12.0 果然报错.
图四
  • 然后我忘了怎么查到这篇文章了Python安装Whl文件_hero的博客-CSDN博客, 给出了这个重要的网址 Python Extension Packages for Windows - Christoph Gohlke. 思路就是包管理器自动装搞不定咱就自己下whl文件手动搞(全手动可能要自己构建whl文件, 笔者目前是搞不懂这个的.)
    • (实际上前面第一个参考链接windows python库wheel文件下载_聿默的博客中第二个网址就给的是这个…又是一个查资料太多不看完的惨痛教训= =|||)
  • 打开网址Python Extension Packages for Windows - Christoph Gohlke, ctrl+f搜索MXNet(图五), 看到有支持cp38的为父很欣慰, 点击下载. 看到requires, 再ctrl+f 搜索numpy如图六. 我没下最新版的也懒得再试了, 反正1.19.5的那个搞成功了.

  • python的依赖是具体到版本的, 有些模块并不支持其依赖的最新版本.
  • python的模块也对python的版本有详细要求.
  • 这模块可能本来是要自己构建编译的, 但是我找到了一个网站有人做好了这些事?
